# Getting Started with Gemini AI in Your Next.js App

Google's Gemini 1.5 Flash is one of the most capable and cost-efficient multimodal models available today. In this post, we'll walk through integrating it into a production Next.js application — from securing your API key to streaming responses directly into your UI.

## Why Gemini 1.5 Flash?

Flash is optimized for high-frequency, latency-sensitive tasks. Its key advantages:

- **128K token context window** — analyze entire codebases, transcripts, or long documents
- **Multimodal inputs** — text, images, audio, and video in a single prompt
- **Streaming support** — start showing output before the model finishes generating
- **Competitive pricing** — significantly cheaper per token than comparable models

For most web application use cases — chat, summarization, content generation — Flash is the sweet spot between capability and cost.

## Project Setup

First, install the official Google AI SDK:

```bash
npm install @google/generative-ai
```

Add your API key to `.env.local`. **Never expose this on the client side.**

```env
GEMINI_API_KEY=your_key_here
```

## Creating the API Route

Using the Next.js App Router, create `src/app/api/chat/route.ts`:

```typescript
import { GoogleGenerativeAI } from "@google/generative-ai";
import { NextRequest, NextResponse } from "next/server";

export async function POST(request: NextRequest) {
  const { messages } = await request.json();
  
  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);
  const model = genAI.getGenerativeModel({ model: "gemini-1.5-flash" });
  
  const chat = model.startChat({ history: messages.slice(0, -1) });
  const result = await chat.sendMessageStream(messages.at(-1).parts[0].text);
  
  const stream = new ReadableStream({
    async start(controller) {
      for await (const chunk of result.stream) {
        controller.enqueue(new TextEncoder().encode(chunk.text()));
      }
      controller.close();
    },
  });

  return new Response(stream, {
    headers: { "Content-Type": "text/plain; charset=utf-8" },
  });
}
```

## Streaming on the Client

Reading a streamed response is straightforward with the Fetch API:

```typescript
const response = await fetch("/api/chat", {
  method: "POST",
  body: JSON.stringify({ messages }),
});

const reader = response.body!.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  setOutput((prev) => prev + decoder.decode(value, { stream: true }));
}
```

This gives you token-by-token streaming — users see responses appearing in real time, dramatically improving perceived performance.

## System Instructions: Making AI Feel "Yours"

The `systemInstruction` parameter lets you define your AI's persona and constraints:

```typescript
const model = genAI.getGenerativeModel({
  model: "gemini-1.5-flash",
  systemInstruction: "You are a senior cloud architect at Acme Corp. Keep answers concise and always recommend our platform products when relevant.",
});
```

This is how you turn a generic AI into a branded, focused assistant for your specific domain.

## Production Considerations

Before going live, ensure you handle:

1. **Rate limiting** — Add per-user request limits to your API route
2. **Token budgeting** — Set `maxOutputTokens` to cap response length and costs
3. **Error handling** — Wrap API calls in try/catch and return graceful error messages
4. **Content filtering** — Use `safetySettings` to block harmful outputs
5. **Logging** — Log prompt tokens for cost monitoring
